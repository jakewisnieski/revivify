# M0 — the walking skeleton

**What this milestone proves:** Revivify can take a landing page, check it against real
published standards, and hand you a plain-language verdict — a trust score, what's wrong,
why it matters, and how to fix it — without you ever reading the code.

**What's deliberately NOT here yet:** the heavy accessibility and performance engines
(axe-core, Lighthouse) and the full 15-rule set — those are **M1**. M0 uses six checks that
can be judged from the page's HTML alone, just to prove the whole pipeline works end-to-end.

---

## Try it yourself (about 2 minutes)

First-time setup (once): `npm install`.

### 1. Watch the guided tour

```bash
npm run walkthrough
```

You'll see Revivify check a raw page (a low score, with every problem explained), then the
same page fixed (a perfect 10/10). This is the whole product idea in one screen.

### 2. Check the example pages yourself

Each example has a **known** result, so you can confirm the tool behaves
([full list](../../examples/README.md)). M0's six checks are now the **`--fast`**
pre-check, so add `--fast` to see exactly the M0 behaviour below (the default
`revivify check` now runs the full M1 audit — see [`m1.md`](m1.md)):

```bash
npm run check -- ./examples/perfect --fast              # 10/10 — Ship-ready
npm run check -- ./examples/starter-slop --fast         # 2/10  — lots wrong
npm run check -- ./examples/missing-alt --fast          # 8/10  — one accessibility miss
npm run check -- ./examples/accidental-noindex --fast   # 8/10  — a "Your call" finding
```

### 3. Check your own page

```bash
npm run check -- ./path-to-your-site --fast   # a folder with an index.html, or…
npm run check -- ./path-to/page.html --fast   # …a single HTML file
```

## What you should see

Running `npm run check -- ./examples/starter-slop --fast` prints a report like this:

```
Revivify checked …/examples/starter-slop/index.html

  Trust: 2/10 — 1 of 6 checks passing

  ✗ Page declares its language
      WCAG 2.2 — 3.1.1 Language of Page (Level A)
      The <html> tag has no lang attribute, so assistive tech can't tell what language…
      → Add a language to the opening <html> tag, e.g. <html lang="en">.  [We'll fix it]

  … (more findings) …

  Not yet ⚠️  — 5 checks to fix. Fix them, then re-run revivify check to watch the score climb to 10.
```

Things to notice — this is the product working as intended:

- **A single trust score** (`X/10`) with a concrete "N of 6 checks passing."
- **Every finding cites a real standard** (WCAG / Google Search Essentials) — authority
  comes from published rules, not an opinion.
- **Plain language + a concrete fix** on every problem. No code-reading needed.
- **Triage labels:** `We'll fix it` (safe), `Just so you know` (info), `Your call`
  (a judgment only you should make — e.g. hiding a page from Google).
- **A clear verdict:** `Ship-ready ✅` only at a perfect 10/10, or `Not yet ⚠️`.

## The six checks in M0

| Check | Standard it cites | Triage |
|---|---|---|
| Page declares its language (`<html lang>`) | WCAG 2.2 — 1.1.1 / 3.1.1 | We'll fix it |
| Page has a `<title>` | Google Search Essentials | We'll fix it |
| Page has a meta description | Google Search Essentials | We'll fix it |
| Page is mobile-friendly (viewport) | Google Search Essentials | We'll fix it |
| Images have alt text | WCAG 2.2 — 1.1.1 | We'll fix it |
| Page isn't accidentally hidden from Google (`noindex`) | Google Search Essentials | **Your call** |

## Acceptance checklist — sign this off

Tick each box by doing the step. When all are ticked, **M0 is doing the right thing** for you
as the end user. If any box won't tick, that's a bug or a product gap — say so and we fix it.

- [ ] **It runs without me touching code.** `npm run walkthrough` completes and reads as
      plain English from top to bottom.
- [ ] **A bad page is caught.** `./examples/starter-slop` scores **2/10** and says
      **Not yet ⚠️**.
- [ ] **A good page passes.** `./examples/perfect` scores **10/10** and says
      **Ship-ready ✅**.
- [ ] **Findings are understandable.** For any failing check I can read what's wrong and what
      to do — without knowing HTML.
- [ ] **Findings cite a real standard.** Every problem names a rule (e.g. "WCAG 2.2 — 1.1.1").
- [ ] **Accessibility is covered.** A missing image alt (`./examples/missing-alt`) is flagged
      and cites WCAG 1.1.1.
- [ ] **SEO is covered.** A missing meta description (`./examples/no-meta-description`) is
      flagged and cites Search Essentials.
- [ ] **Judgment stays with me.** An accidental `noindex` (`./examples/accidental-noindex`) is
      flagged as **Your call**, not silently changed.
- [ ] **The bar is a perfect 10.** Anything less than 10/10 says "Not yet," so I can trust that
      "ship-ready" really means nothing broken was left behind.

**Signed off?** If yes, we move to **M1** (real accessibility + performance engines and the
full 15 rules). If not, note which box failed and what you expected instead.
