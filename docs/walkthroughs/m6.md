# M6 — URL input + design layer (the front door + the ceiling)

**What this milestone proves:** Revivify grows from a standards **gate** into a **design partner** — a functional *floor* plus a design *ceiling*. You can now **paste a live URL** (not just a local build) and get the trust score, and a new **design-advisory panel** offers a grounded second opinion on the visual design — advisory, tier-labeled, and **provably never part of the score.**

**Why it exists:** the URL scan is the **front door** to the steering product ([#28](../decision-log.md)) — the surface a coordinated tester reaches for first. And the deterministic gate only certifies the *floor* (is anything broken?); the design layer starts on the *ceiling* (is this any good?) without ever pretending taste is a standard.

**How it's built:** `check`/`gate`/`ui` accept a live URL, read-only since there's no project to write to (M6.1, FR-1); a deterministic extractor reads the page's real computed styles + a screenshot (M6.2); a grounded critique service sends those to a local vision model through a token-gated endpoint and returns tier-labeled advisory findings (M6.3); and the cockpit renders them in a separate, advisory panel that can't touch the dial (M6.4). This walkthrough is **cockpit-first** — you accept it by *looking* ([#23](../decision-log.md)).

> **One setup note:** the design layer is **availability-gated by design** ([#30](../decision-log.md)). It needs a local vision model reachable via the design env vars. If the model is down or unconfigured, the panel honestly reads *"design analysis unavailable"* and **the deterministic gate is unaffected** — that's an expected state, not a failure. To run the live critique, stand up Qwen3-VL 8B first (see [running-qwen3-vl-locally.md](../explorations/running-qwen3-vl-locally.md)).

## Try it yourself (~5 minutes)

Jake runs Windows **PowerShell**, so this is PowerShell-first. **Every command below is relative to the Revivify repo root — in a fresh window, `cd` there first** (otherwise `npx tsx src/cli.ts …` looks for `src/cli.ts` in whatever folder you happen to be in and fails with `ERR_MODULE_NOT_FOUND`):

```powershell
cd "path\to\Revivify"    # e.g. C:\Users\Natalie\Documents\Claude Code\Revivify
# One-time in this window: allow npm/npx scripts to run.
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

### 1. Paste a URL — the front door (M6.1)

Point the cockpit straight at a live page:

```powershell
npx tsx src/cli.ts ui https://example.com
```

Click **Run check**. Revivify fetches the page, runs the real audit, and shows the trust score — the same dial and cited findings you'd get on a local build. Two things are deliberately different on a **live URL** ([#29](../decision-log.md)):

- **It's read-only.** The intent card reads *"Live URL — read-only … Revivify can't write to a site it doesn't own."* **Apply the safe fixes** and **Accept** are gone — there's no local project to write a fix or an acceptance into.
- **HTTPS becomes real.** On a live `https://` page the transport-security check actually means something (it's not applicable to a `file://` build).

That's the front door: drop in a URL, get the floor. (Fixes and steering come once you point it at a local build you own.)

### 2. The design-advisory panel — the ceiling (M6.2–M6.4)

The richest design example is the styled local demo. **Set the endpoint in the same window *before* you launch `ui`** — the cockpit reads it at startup, so setting it after the server is already running won't reach it, and the panel will honestly say *"design analysis not configured."*

```powershell
# First confirm the model is ready: `ollama list` shows qwen3-vl:8b and Ollama is running
# (see running-qwen3-vl-locally.md). Then, in THIS window, set the endpoint and launch:
$env:REVIVIFY_DESIGN_ENDPOINT = "http://localhost:11434"   # local Ollama, no tunnel
# (Through the token-gated tunnel instead? also set $env:REVIVIFY_DESIGN_TOKEN.)
npx tsx src/cli.ts ui demo-site
```

> If the panel later says *"design analysis unavailable (fetch failed)"* instead, the endpoint **is** set but Ollama isn't answering — start Ollama and confirm `ollama list` shows the model.

Click **Run check** (tick **fast pre-check** for an instant dial). The demo lands **below the bar** on its seeded objective defects — that's the floor, unchanged. Now scroll to **Design suggestions** and click **Get design suggestions**. After ~30–60s (a local vision model is doing real work) the panel fills in, grouped **by tier**:

- **Tier 1 · Objective** and **Tier 2 · Heuristic** — cited (a standard or named research), with a *Learn more* link. The lead finding is the deterministic one, straight from the measured facts: the demo's *competing calls-to-action*.
- **Tier 3 · Taste** — the model's visual read (hierarchy, palette, messaging), each item tagged **"model opinion"** and **carrying no citation** — because it's opinion, not authority ([#31](../decision-log.md)).

Read the closing line every time: *"Advisory only — these design suggestions are never part of the trust score above … a short or empty result is not proof the design is good."* The panel **never oversells** — it can miss a visually-broken page, so its silence is not a pass.

**Watch the dial while you do this.** It does not move. Run the design layer, re-run it, whatever — the trust score is byte-for-byte the same. The design panel is a **separate track** that is architecturally incapable of feeding the score (there's a test that proves it — [#31](../decision-log.md)).

### 3. The honest degrade — "unavailable" is fine

Now see what a tester sees when your box is down. Stop the model (or start the cockpit without `REVIVIFY_DESIGN_ENDPOINT`) and click **Get design suggestions**:

> **Design analysis unavailable.** The deterministic gate and trust score are unaffected — only the advisory design layer is offline.

No error, no blocked verdict, no dinged dial. The floor still fully certifies the page; only the ceiling is quiet. That's the safety property that makes the design layer safe to depend on **and** safe to be absent ([#30](../decision-log.md)/[#31](../decision-log.md)).

## What you should see — and why it's right

- **A URL is a first-class target.** Paste a live page and get the same cited, triaged score as a local build — read-only, because Revivify won't write to a site it doesn't own (M6.1, FR-1; [#29](../decision-log.md)).
- **The design layer grounds the model in real measurements.** The critique reasons over the page's *actual* computed styles + a screenshot, so it can't confabulate pixel values the way an ungrounded vision model does (M6.2/M6.3; experiments B–E).
- **Findings are tier-labeled by how citable they are.** Objective/heuristic carry citations; taste is labeled **model opinion**, never laundered as cited — the honesty line ([#10/#12](../decision-log.md), [#31](../decision-log.md)).
- **The dial is sacred.** The design panel is advisory-only and sits on a separate data path; it cannot move the 10/10 score, and a test enforces it (floor → ceiling boundary, [#31](../decision-log.md)).
- **It never oversells its coverage.** Because it can miss a visually-broken page, the panel says so plainly — "not proof the design is good" — and reserves green for pass/go, so advisory items never read as failures *or* passes ([#11](../decision-log.md), [#31](../decision-log.md)).
- **Unavailable is an expected state.** The model is availability-gated; when it's down the gate is untouched and the panel says "unavailable" — no crash, no false green ([#30](../decision-log.md)).

> **Build-time note (gisted into the record):** decisions [#28–#31](../decision-log.md) and the [PRD §8.3](../prd.md) refinement landed at scope sign-off and read correctly against what shipped. One faithful delta worth naming: the critique service is **endpoint-agnostic** — it speaks to any bearer-gated, Ollama-compatible base URL via `REVIVIFY_DESIGN_ENDPOINT`/`REVIVIFY_DESIGN_TOKEN`/`REVIVIFY_DESIGN_MODEL` — so #30's "token-gated Cloudflare Tunnel" is *one* deployment of a transport-neutral seam, not a hard dependency. The scan→steer bridge remains the Phase-2 load-bearing UX ([#28](../decision-log.md)).
