# M5 — Demo + polish (the shareable product)

**What this milestone proves:** Revivify is now **credible and demonstrable in an interview.** There's a realistic landing page that *looks shippable but fails silently*, a cockpit **hero GIF** in the README that tells the whole story in ten seconds, the config toggles actually *work*, and the "done" gate is locked by an end-to-end test.

**Why it exists:** the product only lands if someone can *see* it working on something that looks real — not a toy fixture. M5 makes the pitch legible (PRD §10; decisions [#24–#26](../decision-log.md)).

**How it's built:** a styled `demo-site/` seeded with the invisible defects (M5.1), a recorded cockpit GIF + rewritten README (M5.2), `.revivify.yaml` **rule toggles** consumed honestly by `check`/`gate` (M5.3, FR-10), and an e2e **block-mode gate** test (M5.4). This walkthrough is **cockpit-first** — you accept it by *looking*.

## Try it yourself (~4 minutes)

Jake runs Windows **PowerShell**, so this is PowerShell-first. From the repo root:

```powershell
# One-time in this window: allow npm/npx scripts to run.
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

### 1. The demo page — "it looks done, but it fails silently"

Point the cockpit at a **copy** of the demo (the fix step in beat 2 edits the page, so we work on a throwaway copy):

```powershell
$DEMO = "$env:TEMP\revivify-m5-demo"
Remove-Item -Recurse -Force $DEMO -ErrorAction SilentlyContinue
Copy-Item .\demo-site $DEMO -Recurse
Remove-Item "$DEMO\.revivify.yaml","$DEMO\README.md" -ErrorAction SilentlyContinue
npx tsx src/cli.ts ui $DEMO
```

In the browser that opens, click **Run check** (or tick **fast pre-check** first for an instant run). You land **Not yet** on a page that *looks* completely shippable. The cockpit names exactly what's wrong — each cited to its standard:

- **Images have alt text** ✗ (WCAG 1.1.1), **Page declares its language** ✗ (WCAG 3.1.1), **Page has a meta description** ✗ — all *"we'll fix it."*
- **Page isn't accidentally hidden from search engines** — a leftover `noindex`, surfaced as **your call.**

That's the whole thesis in one screen: a non-dev would ship this page, and Google + screen-readers would quietly hate it.

### 2. Own the fix — click to ship-ready

In the **Plan** card, click **Apply the safe fixes**. Revivify writes the "we'll fix it" fixes to the page itself — but only values it can **source honestly**: `lang="en"`, a meta description from the page's own lede copy, and `alt` pulled from the hero image's **own declared name** (the SVG's `aria-label`). It re-checks, and the dial **climbs to 9 / 10** — every objective check now passing. (Anything it can't source honestly is left for your coding agent, never guessed.)

One item remains: the **your call** `noindex`. In the **Your call** card, click **Accept this — it's intentional**, type a reason (e.g. *"Staging preview — hidden until launch."*), **Confirm** — and the dial **clears to 10 / 10 · Ship-ready**, with the item still visible, marked **accepted**, showing your reason. No code-reading, no YAML editing. (The README GIF shows this same accept beat.)

### 3. A toggle you set actually changes the gate (M5.3 / FR-10)

```powershell
$T = "$env:TEMP\revivify-m5-toggle"
Remove-Item -Recurse -Force $T -ErrorAction SilentlyContinue
Copy-Item .\examples\missing-alt $T -Recurse
"threshold: 10`nenforcement: warn`nrules:`n  img-alt: false" | Set-Content "$T\.revivify.yaml" -Encoding utf8
npx tsx src/cli.ts check $T
```

`examples/missing-alt` normally scores 9/10 on the one alt miss. With `img-alt: false`, the report shows **"Disabled in .revivify.yaml, not scored: Images have alt text"** — the check is **dropped from the denominator, not faked as a pass.** That's the honesty line: a toggle you set is real, and what you turned off stays visible.

## What you should see — and why it's right

- **The demo looks shippable but isn't.** A styled page (real CSS, hero, pricing, reviews) that a non-dev would ship — with only the four *invisible* seeds failing. Colours clear WCAG AA contrast, so nothing else is flagged ([#24](../decision-log.md)).
- **The cockpit is the hero.** The README leads with a GIF of the real `revivify ui` taking a page **9 → 10** by accepting the your-call — the visual differentiator, plus a runnable CLI quickstart ([#25](../decision-log.md)).
- **The cockpit owns the safe fix — honestly.** *Apply the safe fixes* writes the "we'll fix it" batch on approval, but only values it can **source from the page/assets** (`lang`, the description from the page's own copy, `alt` from the image's own declared name); anything it can't source is **left for the agent**, never a fabricated `alt` ([#27](../decision-log.md), refining [#20](../decision-log.md); honesty line [#10/#12](../decision-log.md)).
- **A "your call" is resolved, not faked.** Accepting keeps it visible with its reason and never a green ✓; the dial clears only when every objective check passes *and* every your-call is resolved ([#18](../decision-log.md)).
- **Config toggles are honest.** A disabled check drops from the score like a not-applicable one, but is named as *disabled by config* — never a silent green ([#26](../decision-log.md), [#10/#12](../decision-log.md)).
- **The gate can't silently regress.** An end-to-end test locks block-mode: below the bar → the hook keeps the agent going (exit 2); at the bar → done is allowed (exit 0).
